# Audio Player - AI Agent Context

## Challenge Info
- **Day:** 78 of 100
- **Design:** See `design.png` in this folder
- **Starter file:** `index.html`

## User Story

As a user, I want to play and control audio tracks directly from a visually appealing player, so that I can easily listen to music or other audio content.

## Acceptance Criteria

- Display the current track name and artist below the track's visual representation.
- Include basic playback controls, such as play/pause and a progress bar.
- Show the track's total duration and the current playback time.
- Provide a volume control option within the player.
- Ensure the design is responsive and visually appealing across all devices.

## Technical Notes

- Use any framework or vanilla HTML/CSS/JS
- The design mockup (`design.png`) is the visual target
- Icons, fonts, and images need to be sourced (use Google Fonts, Font Awesome, Unsplash, etc.)
- Make it responsive across mobile, tablet, and desktop
- Add interactivity where it makes sense

## Suggested Approach

1. Analyze `design.png` for layout structure, colors, typography, and spacing
2. Build the HTML skeleton with semantic elements
3. Style with CSS to match the design
4. Add JavaScript for interactivity
5. Test responsiveness and accessibility
